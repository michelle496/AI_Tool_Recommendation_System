# AI_Tool_Recommendation_System
# 🤖 场景化 AI 工具推荐助手 (AI-Tool-Recommender-Agent)
### 🌟 项目简介
* 由于AI工具更新快、官方榜单用词专业，对于新手不好理解，开发场景化AI工具推荐助手。
* 可理解用户需求（如做PPT，语音转文字），并推荐Top5大众好用的最新AI工具（附官网链接，可直接体验使用），解决用户“找不到”、“难理解”、“跟不上更新”的问题。
* 同时，为了防止AI幻觉，在返回Top 5推荐后，将参考文档（即搜索到的AI工具推荐等网页链接）一并返回，让用户可以在AI推荐的5个工具之外探索更多选择。

-----

### 🚀 核心功能亮点

  * **🧠 意图对齐与动态加权**：系统通过 LLM 预分析用户场景（如：办公、科研、开发），自动分配“榜单、评测、部署、体验”四维度的搜索权重，拒绝无思考通用检索。
  * **🔍 定向配额检索**：接入 Tavily 搜索引擎，根据计算出的权重配额进行精准抓取，确保信源的多样性与专业性。
  * **🛡️ 防幻觉双层交付**：针对 LLM 易编造链接的痛点，采用“结论先行+证据溯源”模式。每一个推荐工具均需在原始网页中找到对应出处，彻底杜绝虚假链接。
  * **📊 结构化专家报告**：输出包含 Top 5 核心推荐表格及分维度的参考文献，满足用户从“快速决策”到“深度调研”的全方位需求。

-----

### 🛠️ 技术架构

1.  **意图识别层**：识别需求所属垂直领域。
2.  **战略规划层**：生成维度配额及针对性搜索词。
3.  **执行层**：多线程或定向配额检索。
4.  **精炼层**：上下文摘要提取与链接有效性对齐。
5.  **展示层**：Markdown 自动化渲染。

### 🧪 自动化评测体系 (Evaluation System)

为了验证 Agent 在不同需求场景下的决策科学性，本项目建立了一套完整的**“决策-评测-对齐”**体系，将黑盒的 AI 推理过程转化为可量化的工程指标。

### 1. 黄金评测集构建 (Golden Dataset)
项目内置了一份经过专家人工标注的测试集（`动态权重分布评测集.xlsx`），包含 100+ 条真实的用户原始诉求。
* **Question**: 覆盖从“学术论文润色”到“本地代码库构建”的各类长尾需求。
* **Ideal_Weight_Distribution**: 专家预设的理想搜索配额（例如：学术类加重“深度评测”，工具类加重“工程部署”）。

| 场景分类 | 用户原始需求 (Question) | 理想配额分布 (榜单:评测:部署:体验) | 决策逻辑 (Logic) |
| :--- | :--- | :--- | :--- |
| **办公效率** | "想找一个能自动把会议录音转成整理好的会议纪要的工具，要支持中文。" | **2 : 2 : 1 : 0** | 重点在于看哪家工具准确度高（评测）以及好不好找（榜单），部署只需指导如何登录。 |
| **开发者工具** | "帮我找个 VS Code 插件，能直接把自然语言转成 SQL 语句，不用配置环境。" | **1 : 2 : 2 : 0** | 用户强调“不配置环境”，因此需要重点搜索“一键安装指南”（部署）和生成准确度（评测）。 |
| **多模态创作** | "想找个 AI 能帮我给短视频配音，要声音听起来像真人，不要那种机械音。" | **2 : 1 : 0 : 2** | 声音是否真实非常主观，因此需要大量“用户反馈”（体验）和“横向对比”（榜单）。 |
| **学术科研** | "推荐几个能自动总结 PDF 文献核心观点的工具，最好能导出思维导图。" | **1 : 3 : 1 : 0** | 学术场景对准确性要求极高，配额向“深度评测”倾斜，看其总结是否会产生幻觉。 |
| **垂直咨询** | "有没有 AI 能帮我初步分析合同里的法律风险？我是小微企业主，不想找律师。" | **2 : 2 : 0 : 1** | 法律工具需要权威背书（榜单）和功能边界测试（评测），防止产生错误法律建议。 |

### 2. 决策对齐算法 (Alignment Score)
通过核心函数 `calculate_distribution_adaptation`，系统会自动执行以下评测逻辑：
1. **决策抓取**：Agent 接收 Question 后，预先生成其认为最优的 [场景分类] 与 [维度权重]。
2. **相似度计算**：计算 Agent 生成的权重分布与专家标注分布的交集（Intersection）。
3. **得分转换**：
   $$对齐得分 = \frac{\sum min(实际配额, 理想配额)}{总配额} \times 100\%$$

### 3. 自动化评测报告示例
运行评测后，系统会自动导出 `Agent决策对齐报告.xlsx`，量化每一次策略调整的效果：

| 原始需求 (Query) | 判定场景 | 理想分布 (专家) | Agent 实际决策 | 对齐得分 |
| :--- | :--- | :--- | :--- | :--- |
| "我想写周报，求推荐工具" | 办公效率 | 榜单:2, 体验:3 | 榜单:2, 体验:3 | **100%** |
| "本地部署 DeepSeek 教程" | 开发者工具 | 部署:4, 评测:1 | 部署:3, 榜单:2 | **80%** |
| "对比各家大模型 API 价格" | 垂直领域咨询 | 评测:3, 榜单:2 | 榜单:5, 体验:0 | **40%** |

### 4. 评测驱动的迭代产出
* **发现问题**：通过早期评测报告，发现 Agent 在面对模糊需求时倾向于全量搜索“榜单”，导致在专业领域搜索深度不足。
* **优化举措**：重构了 `define_search_strategy` 的提示词逻辑，引入动态加权策略。
* **量化成果**：经过迭代，Agent 的 **场景适配度从40%提升至80%，有效采纳率（停留>5min）从30%翻倍至60%**，大幅增强了搜索结果的专业对位。
